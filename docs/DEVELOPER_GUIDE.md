# Developer Guide

## Extend the System

Keep changes modular. Agents should own reasoning responsibilities, while tools should own reusable deterministic operations.

## Add Agents

Add a file in `app/agents/`, expose a node in `app/graph/nodes.py`, and wire it into `app/graph/workflow.py`.

## Add Tools

Add functions under `app/tools/`. Tools should accept explicit parameters and return typed or simple values. Avoid hidden environment coupling except in configuration.

## Change Graph Edges

Edit `app/graph/workflow.py`. If a route is conditional, implement a route function in `app/graph/nodes.py` and add tests that exercise both branches.

## Coding Standards

- Typed Python.
- Small modules.
- No hardcoded secrets.
- Deterministic mock mode.
- Meaningful comments only where useful.
- Tests for every new agent or route.

## Debugging

Use agent notes in `state.notes` to inspect execution. Use SQLite run metadata with `/runs/{run_id}` or `MemoryStore().get_run(run_id)`.

## Report Generation

The primary final deliverable is generated by `app/tools/latex_report_tools.py`. Keep LaTeX content deterministic and source-backed. If you add report sections, update both the LaTeX tool and the tests that verify report artifacts.

## Presentation Generation

`PresentationAgent` uses `python-pptx` when available and a minimal OpenXML fallback otherwise. Keep the Markdown deck in sync with the PPTX slide spine.
